Tab5 手势传感器 — I2C 总线冲突

ESP32-P4 共 3 条 I2C 控制器(2×HP + 1×LP)。问题:键盘和手势抢同一条对外总线。

现状:三条 I2C 怎么分

HP I2C_NUM_1内部 · G31/G32
触摸 0x55IMU 0x68 RTC 0x32电源/INA226 0x41 IO扩展 0x43/0x44音频 0x10/0x40 锁死 · 不能动
HP I2C_NUM_0唯一对外总线
键盘 A164 0x6D · 要 G0/G1 ⚔︎ 抢 手势 PAJ7620 0x73 · 要 G53/G54
LP I2C第3条
只能接低位 GPIO 0–15 G0/G1 ✓ 够得到 G53/G54 ✗ 够不到(Port A)
冲突本质: 同一个 I2C 外设(NUM_0)在任一时刻只能绑一对引脚。键盘在 ExtPort1 的 G0/G1,手势在 Port A 的 G53/G54 —— 两者都要 NUM_0,但引脚不同, 没法同时各占。kbd.cpp 里 Wire.begin(0,1) 就是把 NUM_0 抢到了 G0/G1。 LP 这条第三总线够不到 Port A 的高位引脚,救不了手势,但能接键盘(G0/G1 是低位)。

四个方案

① 只上手势 推荐

NUM_0 → 手势 0x73 @ G53/G54 ✅ NUM_1 → 内部(不变) A164 键盘 I2C → 关闭 USB-A 键盘 → 不受影响(走 USB,另一条路)
  • 最干净、最稳,零冲突
  • USB-A 插的键盘照常用
  • 暂时没有 A164 卡扣键盘当"第二键盘"
  • A164 的 Del=录音 可由手势(推/挥)替代

② 分时复用 有风险

每轮循环: 切引脚→G53/G54 读手势 切引脚→G0/G1 读键盘 一条 NUM_0 轮流伺候两边
  • 键盘 + 手势都保留
  • 每轮重配总线引脚,hacky
  • M5UnitUnified 管 Ex_I2C 与裸 Wire 混用易出错
  • 时序/抖动风险,调试成本高

③ Grove 分线/PaHub 需硬件

键盘 + 手势 接到同一条总线 同 NUM_0 · 同引脚 · 不同地址 (0x6D 与 0x73 共存一条 bus)
  • 软件最简单(I2C 本就支持多设备)
  • 键盘 + 手势都保留
  • 需要 Grove Hub / 把手势接到键盘那条线

④ 键盘挪到 LP I2C 待验证

LP I2C → 键盘 0x6D @ G0/G1(低位 ✓) NUM_0 → 手势 0x73 @ G53/G54 NUM_1 → 内部
  • 理论最干净:三设备各占一条
  • 键盘 + 手势都保留
  • Arduino 对 LP_I2C 支持不确定,要先研究
键盘 A164(I2C 0x6D) 手势 PAJ7620(I2C 0x73) 内部芯片(锁死) 推荐
